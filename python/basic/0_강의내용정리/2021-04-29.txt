라이브러리 : library
개발자가 모듈의 함수를 실행하여 사용하는것. 정상적인 제어를 하는 모듈. 하지만, 함수와 모듈 등을 많이 모아 둔 것을 라입러리라고 함. 

프레임워크 : framework
모듈이 개발자의 함수를 실행하여 사용하는것. 제어역전이 발생하는 모듈. 프레임워크, 말 그대로 어떤 틀에 맞게끔 일하는 것을 말함. 어떤것을 작업하는데 모두가 같은 결과물을 만들게 틀을 주는것. 즉, 머신러닝의 모델이나 웹개발 틀인 장고 등이 프레임워크에 해당함.

제어역전 : IoC. Inversion of Control
개발자가 모듈의 함수를 호출하는 것이 일반적ㅇ니 제어 흐름인데, 이와 반대로 개발자가 만든 함수를 모듈이 실행하는 ㄱ서으 의미함.
사용자 정의 함수를 모듈이 실행하게 끔 하는것. 우선순위를 의미하는듯. 개발자의 모듈함수를 호출하는것이 1순위인데 내가 만든 함수를 1순위로 만들어 사용하게끔 하는걸의미. 잘 사용안함.
